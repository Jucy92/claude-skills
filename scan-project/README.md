# scan-skills

프로젝트 전체 구조를 스캔하여 지식 파일을 만들고, 기능 구현 질문에 정확하게 답해주는 Claude Code 스킬입니다.

## 이 스킬의 특별한 점

일반적으로 Claude에게 "이 기능 어떻게 구현해?"라고 물으면 소스 파일을 처음부터 전부 읽어야 합니다. 파일이 많을수록 시간이 걸리고, 이미 읽은 내용을 다음 대화에서 다시 읽어야 하는 문제도 생깁니다.

이 스킬은 프로젝트를 한 번 스캔해 **키워드 인덱스**를 만들어 둡니다. 이후 구현 질문이 들어오면 Claude가 인덱스에서 관련 섹션만 골라 읽고 현재 구조에 맞는 정확한 답변을 제공합니다.

### 어떻게 사용하나요?

프로젝트를 열고 처음 한 번만 실행하면 됩니다:
```
/scan-skills
```

이후에는 평소처럼 질문하면 됩니다:
```
"위치 기반 거리 측정 기능 어떻게 추가해?"
"결제 취소 API 어디에 구현하면 돼?"
```

Claude가 자동으로 관련 파일 섹션만 찾아 읽고 답변합니다.

## 무엇을 할 수 있나요?

**프로젝트 구조 스캔 (scan 모드)**
- 백엔드(Spring Boot)와 프론트엔드(Vue/Node) 구조를 자동 감지합니다
- 엔티티 필드, 서비스 메서드, API 엔드포인트를 추출합니다
- 기술 스택과 미구현 영역(Redis, Kafka 등)을 파악합니다
- 4개의 지식 파일과 파생 스킬을 자동으로 생성합니다

**키워드 기반 탐색 (navigate 모드)**
- 질문에서 키워드를 추출하여 관련 섹션만 선택적으로 읽습니다
- 동의어도 매칭합니다 (예: "위치" → GPS, 좌표, 거리, latitude 모두 매칭)
- 현재 실제 구현 상태를 기준으로 가이드를 제공합니다
- 프로젝트에 없는 기능이면 "미구현 영역"임을 먼저 알려줍니다

**자동 규칙 등록**
- CLAUDE.md에 탐색 규칙을 자동으로 추가합니다
- 이후 모든 대화에서 소스 파일을 무작위로 읽는 대신 지식 파일을 먼저 확인하게 됩니다

## 시작하기

### 필요한 것

- Claude Code에서 프로젝트를 열어야 합니다
- 특별한 설치나 설정은 필요하지 않습니다

### 처음 실행

```
/scan-skills
```

스캔이 완료되면 `.claude/` 폴더에 지식 파일이 생성됩니다. 이후에는 아무 질문이나 하면 됩니다.

### 업데이트

새 도메인이나 기능을 추가한 경우:
```
/scan-skills update
```

지식 파일을 완전히 새로 만들고 싶은 경우:
```
/scan-skills reset
```

## 작동 원리

**scan 모드는 8단계로 작동합니다:**

**1단계: 모드 결정**
`.claude/PROJECT_SCAN.md`가 있으면 갱신 모드, 없으면 최초 생성 모드로 동작합니다. `reset` 인수를 주면 전체 재생성합니다.

**2단계: 프로젝트 감지**
`backend/build.gradle`과 `frontend/package.json` 유무로 기술 스택을 자동 판별합니다.

**3단계: 백엔드 스캔**
엔티티 필드, 서비스 public 메서드, 컨트롤러 API 매핑, application.yml 기술 스택을 추출합니다.

**4단계: 프론트엔드 스캔**
라우터 설정, 뷰 컴포넌트, API 호출 패턴을 추출합니다.

**5단계: 지식 파일 생성**
수집한 데이터를 4개의 파일로 정리합니다. 자세한 내용은 아래 [생성되는 파일](#생성되는-파일) 섹션을 참고하세요.

**6단계: 파생 스킬 생성**
파일 수정 시 사이드 이펙트를 추적하는 `verify-impact` 스킬을 자동으로 생성합니다.

**7단계: 연관 파일 자동 편집**
`manage-skills`, `verify-implementation`, `CLAUDE.md`에 필요한 항목을 자동으로 추가합니다.

**8단계: 완료 보고서 출력**
생성된 파일 목록, 스캔된 도메인 수, API 엔드포인트 수를 출력합니다.

---

**navigate 모드는 질문이 들어올 때 자동으로 작동합니다:**

"어떻게 구현해?", "어디에 추가해?" 같은 질문을 감지하면 자동으로 활성화됩니다.

**1단계:** 질문에서 키워드를 추출합니다.

**2단계:** `KEYWORD_INDEX.md`에서 관련 도메인과 섹션을 찾습니다.

**3단계:** 해당 섹션만 읽고 현재 구조 기반으로 답변합니다.

**4단계 (키워드 미매칭):** 동의어 검색을 시도하고 `#meta` 섹션에서 기술 스택을 확인합니다.

**5단계 (신규 기능):** 프로젝트에 없는 기능임을 명시하고, 기존 패키지 구조에 맞는 구현 방법을 웹에서 찾아 제안합니다.

## 생성되는 파일

스캔이 완료되면 `.claude/` 폴더에 다음 파일들이 생성됩니다:

- `.claude/KEYWORD_INDEX.md` - 도메인과 키워드를 연결하는 경량 인덱스. 매 대화마다 자동으로 로드됩니다
- `.claude/scan-backend.md` - 백엔드 도메인별 엔티티, 서비스, API 상세 정보
- `.claude/scan-frontend.md` - 프론트엔드 라우터, 뷰, API 호출 상세 정보
- `.claude/PROJECT_SCAN.md` - 전체 프로젝트 지식 맵
- `.claude/skills/verify-impact/SKILL.md` - 파생 스킬 (파일 수정 시 사이드 이펙트 추적)

## 현재 제한사항

- Spring Boot + Vue/Node 구조에 최적화되어 있습니다. 다른 스택은 스캔 결과가 불완전할 수 있습니다
- navigate 모드는 KEYWORD_INDEX.md가 있어야 동작합니다. 먼저 scan 모드를 실행해야 합니다
- 외부 패키지(`node_modules`, `.gradle`)는 탐색 대상에서 자동 제외됩니다
- 테스트 파일(`src/test/`)은 도메인 분석에서 제외됩니다

## 더 알아보기

스킬의 내부 동작 방식이나 커스터마이징 방법이 궁금하다면 `.claude/skills/scan-skills/SKILL.md` 파일을 참고하세요. 각 단계의 상세한 명령어와 파일 생성 포맷이 포함되어 있습니다.
